## 🔲 당신이 해야 할 것

### 1. Jira 환경 변수 (.env)

- 프로젝트 루트(`files` 폴더)에 **`.env`** 파일을 만드세요. (`.env.example`을 복사해서 쓰면 됩니다.)
- 아래 세 가지를 채우세요.

| 변수 | 설명 |
|------|------|
| **JIRA_BASE_URL** | Jira 사이트 URL (예: `https://회사도메인.atlassian.net`) |
| **JIRA_EMAIL** | Jira 로그인 이메일 |
| **JIRA_API_TOKEN** | [Atlassian API 토큰](https://id.atlassian.com/manage-profile/security/api-tokens)에서 발급한 토큰 |

- 대신 **ATLASSIAN_URL**, **ATLASSIAN_API_TOKEN** 도 사용 가능합니다.

### 2. 한 번에 실행 (캘린더처럼)

- **경로:** `/Users/jaewoo.hong/Downloads/files`
- **의존성 설치:**  
  `npm install`

(또는 터미널 두 개에서: 하나는 `npm run api`, 다른 하나는 `npm run dev`.)

### 3. (선택) 다른 Jira 프로젝트 쓰기

- 기본은 **프로젝트 키 BA**입니다. 프론트가 `/api/bugs?project=BA` 로 요청합니다.
- 다른 프로젝트를 쓰려면 `useBugData.ts` 안의 `project=BA` 를 원하는 키로 바꾸면 됩니다.

---

### 4. 500 오류 나올 때 확인할 것

- **API 서버를 띄운 터미널**에서 에러 로그 확인 (이제 `npm run api` / `npm run dev:all` 로 이 프로젝트에서 실행).
- 로그에 `401`, `403`, `token`, `JIRA_BASE_URL`, `필요합니다` 등이 있으면 → **1번(.env)** 에 Jira URL·이메일·API 토큰이 제대로 들어갔는지 확인.
- Jira 프로젝트 키 **BA**가 맞는지, 사용 중인 계정이 그 프로젝트 접근 권한이 있는지 확인.

### 5. (선택) 다른 API 서버를 쓰는 경우

- 이 프로젝트의 `server/index.js` 대신 **외부 백엔드**를 쓸 때만, 프론트 루트에 `.env`에 다음을 넣습니다.  
  `VITE_API_BASE_URL=http://실제백엔드주소:포트`  
  그 다음 `npm run dev` 다시 실행.
